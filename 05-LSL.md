# Bridge 05 — Lab Streaming Layer (BCI / neuroscience)

> **Prerequisite:** `00-FRAMEWORK.md`.

**Priority:** high. **Safety ceiling:** `ADVISORY` (read-mostly). Stimulation outputs are gated behind a `StimulationSafetyGateNeuron` that already exists in the repo (`worker/.../bci/StimulationSafetyGateNeuron.java`); the bridge wires into it.

## 1. Domain context

[Lab Streaming Layer](https://labstreaminglayer.org/) is the open-source middleware ecosystem used in BCI and physiology research. It standardises streaming, time-synchronisation, and recording of neural, physiological, and behavioural data across heterogeneous acquisition devices (EEG amplifiers, EMG, eye-trackers, motion-capture, custom hardware).

LSL is unusual: there is **no central broker**. Devices announce streams via mDNS multicast; clients discover and pull them. The bridge is therefore an LSL "inlet" client. The Java binding is `liblsl-Java`.

For Jneopallium, LSL is the natural first BCI bridge because:
* It works against synthetic streams generated by free tools (BrainVision Recorder, OpenViBE, even a Python script) — no invasive or proprietary hardware needed.
* The repo's `bci/` signal package already contains the destination types (`NeuralSpikeSignal`, `LFPSignal`, `ECoGSignal`, `IntentSignal`, `DriftEstimateSignal`, `SeizureRiskSignal`, `AgencyLossSignal`, `StimulationCommandSignal`, `ChargeAccumulationSignal`, `CalibrationSignal`, `ThermalSignal`, `SensoryFeedbackSignal`).
* It enables the affect modules (`affect/InteroceptiveSignal`, `AffectStateSignal`) when paired with HRV / GSR / temperature streams.

## 2. Maven dependency

```xml

```

Native libraries (`liblsl.so` / `liblsl.dylib` / `lsl.dll`) are shipped inside the jar by recent versions; the maven artifact handles platform detection. Verify before merging.

## 3. Why this bridge is read-mostly

Reading EEG/physiology and emitting typed signals is safe — the data is observational. Writing back — i.e. issuing a `StimulationCommandSignal` to a closed-loop neurostimulator — is not. The repo already has `StimulationSafetyGateNeuron`; the bridge spec only requires that no stimulation output bypasses it.

The aggregator in this bridge therefore writes to **LSL outlets that downstream stimulator software subscribes to**. There is no direct path from this bridge to electrodes. Subscribing software (the actual stimulator driver) is a separately certified component.

## 5. Signal mapping

All target signals already exist; this is purely a mapper bridge.

| LSL stream type (typical) | Channels | Jneopallium signal |
|---|---|---|
| `EEG`, 250–2048 Hz | per-electrode | `LFPSignal` (per-channel windowed) |
| `Spikes`, irregular | unit id | `NeuralSpikeSignal` |
| `ECoG`, 1–10 kHz | electrode | `ECoGSignal` |
| `EMG` | per-muscle | `ProprioceptiveSignal` (or `bci.SensoryFeedbackSignal` depending on use case) |
| `HRV`, 1–10 Hz | RR-interval | `affect/InteroceptiveSignal` |
| `GSR` (skin conductance) | scalar | `affect/InteroceptiveSignal` |
| `Eye` (gaze, pupil) | x/y/diam | `affect/AppraisalSignal` (attention proxy) |
| `Temperature` | scalar | `bci/ThermalSignal` |
| `Markers` (event marker stream) | string | bridge-level event; emit `CalibrationSignal` if marker matches a calibration cue |

Egress (stimulator-driver-consumed):

| Jneopallium signal | LSL outlet |
|---|---|
| `StimulationCommandSignal` (only after the safety gate) | named outlet `Jneopallium-Stim-Advisory`, marker stream |
| `IntentSignal` | named outlet `Jneopallium-Intent`, irregular |
| `SeizureRiskSignal` | named outlet `Jneopallium-Risk`, 10 Hz |

## 6. Configuration

```yaml
discovery:
  resolveTimeoutMs: 2000
  expectedStreams:                 # bridge fails fast if any not found
    - "OpenViBE-EEG-256Hz"
    - "Polar-HRV"

reads:
  - bindingId: "EEG-MAIN"
    streamName: "OpenViBE-EEG-256Hz"
    streamType: "EEG"
    channels: ["Cz","Fz","Pz","Oz","C3","C4"]
    chunkLengthSamples: 64
    targetSignal: "LFP"             # → LFPSignal
    signalTagPrefix: "BCI.EEG"

  - bindingId: "HRV"
    streamName: "Polar-HRV"
    streamType: "HRV"
    targetSignal: "INTEROCEPTIVE"
    signalTag: "BCI.HRV"

writes:
  - bindingId: "STIM-ADVISORY"
    outletName: "Jneopallium-Stim-Advisory"
    type: "Markers"
    nominalSrate: 0                 # irregular
    signalTag: "BCI.STIM.ADVISORY"

audit:
  localAuditFile: "/var/log/jneopallium/lsl-audit.jsonl"

perTagSafetyMode:
  STIM-ADVISORY: ADVISORY            # AUTONOMOUS rejected by config validator
```

## 8. Phase plan

| Phase | Goal |
|-------|------|
| 1 | Read-only against synthetic LSL streams (Python `pylsl` test publisher). Time-sync via LSL `local_clock`. Per-channel decimation. |
| 2 | Outlet write for advisory markers. Wire through existing `StimulationSafetyGateNeuron`. |
| 3 | **Not pursued.** Closed-loop neurostimulation requires medical-device-grade certification outside the scope of this project. |

## 9. Bridge-specific scenarios

| # | Scenario | Setup | Expected |
|---|----------|-------|----------|
| **S7** | Discover and bind | Synthetic publisher emits an `EEG` stream named `OpenViBE-EEG-256Hz` | Bridge resolves within `resolveTimeoutMs`; emits `LFPSignal` chunks at the configured rate |
| **S8** | Time sync | Synthetic publisher emits known monotonic timestamps | Emitted signal `timestamp`s match the LSL stream `time_correction`-adjusted values within 2 ms |
| **S9** | Stream disappears | Kill the publisher | Cache stops updating; advisory event `AlarmSignal(LSL_STREAM_LOST)` emitted within one tick |
| **S10** | Channel mismatch | Stream provides 32 channels but config requests `["Cz","Fz","X9"]` and `X9` does not exist | Config-load fails fast with the missing channel name |
| **S11** | Outlet write | StimulationSafetyGateNeuron emits a permitted `StimulationCommandSignal` | Bridge publishes a marker on `Jneopallium-Stim-Advisory`; audit `verdict=APPLIED` |

## 10. Risks

| # | Risk | Mitigation |
|---|------|------------|
| R1 | Native library ABI mismatch | Maven artifact bundles platform binaries; test on linux64 + osx-arm64 + win64 in CI. |
| R2 | Buffer overflow when ML pipeline lags | Per-stream ring buffer with explicit max-lag; oldest dropped, audit entry. |
| R3 | Privacy of physiological data | Document local-only, ephemeral storage; no cloud egress; explicit consent capture in the operator runbook. |
| R4 | Subject identifier in stream metadata | Bridge MUST strip / pseudonymise subject IDs before logging; redaction is part of the mapper, tested in S11+. |
