# Module F — Adaptive Tutoring & Student Cognitive State Modelling

## Abstract

The tutoring module is a domain-specific application of the jneopallium framework to intelligent tutoring systems. It adds 10 new signals and 14 new neurons spread across layers 0 / 2 / 3 / 4 / 5 / 7, with no changes to the core `INeuron` / `ISignal` / `ISignalProcessor` interfaces. It composes the previously-merged affect, curiosity and sleep extension modules; spaced repetition is driven by `HippocampalReplayNeuron` during sleep, moment-to-moment state by `FlowStateNeuron`, concept mastery by Bayesian Knowledge Tracing (Corbett & Anderson 1995), and the pedagogical planner by a zone-of-proximal-development scoring function (Vygotsky 1978). Default is `enabled: false`.

## Design principles

- **Formative, not summative.** No high-stakes grading; the tutor only adjusts content and pacing.
- **Multi-timescale by construction.** Moment-to-moment (L0 / L2), within-session (L3 / L4), cross-session (L3 forgetting curve + sleep replay) all coexist without bolted-on scheduling.
- **Safety and fairness first.** `WellbeingGuardNeuron` specialises `HarmGateNeuron`; `FairnessNeuron` specialises `EthicalPriorityNeuron` but never modifies its weights (enforced by the safety invariants in `CLAUDE.md` §9.3).
- **Privacy posture.** Instance-per-learner by default; no cross-learner leakage.

## Signals

| Signal | Loop / Epoch | Purpose |
|---|---|---|
| `ItemPresentationSignal` | 1 / 1 | Item shown to learner. |
| `ResponseSignal` | 1 / 1 | Learner response + latency. |
| `EngagementSignal` | 1 / 2 | Fused attention estimate. |
| `AffectObservationSignal` | 1 / 2 | Coarse behavioural affect → affect module. |
| `MasteryUpdateSignal` | 2 / 3 | BKT posterior update. |
| `ContentRecommendationSignal` | 1 / 3 | ZPD-scored next item. |
| `HintSignal` | 1 / 2 | Graduated hint. |
| `ScaffoldingSignal` | 1 / 2 | Outline / analogy / worked steps / reminder. |
| `ReviewScheduleSignal` | 2 / 5 | Next review tick (Ebbinghaus + SM-2). |
| `InterventionSignal` | 2 / 1 | Break / encourage / redirect / escalate. |

## Neurons

| Neuron | Layer | Role |
|---|---|---|
| `ResponseObserverNeuron` | 0 | Accuracy accumulator + error delta. |
| `EngagementSensorNeuron` | 0 | Multi-modal fusion (click, dwell, camera, mic). |
| `AffectObserverNeuron` | 0 | Coarse affect inference from behaviour. |
| `FlowStateNeuron` | 2 | Flow / boredom / frustration / overload classifier. |
| `FatigueNeuron` | 2 | Session-duration & error-drift break trigger. |
| `ConceptMasteryNeuron` | 3 | BKT per concept. |
| `PrerequisiteGraphNeuron` | 3 | Concept DAG + "eligible next" query. |
| `ForgettingCurveNeuron` | 3 | Ebbinghaus + SM-2 spacing. |
| `ZPDPlanningNeuron` | 4 | Vygotsky-style candidate scoring. |
| `HintGenerationNeuron` | 4 | Graduated hint ladder. |
| `ScaffoldingNeuron` | 4 | Scaffolds when overloaded / frustrated. |
| `ContentSelectionNeuron` | 5 | Softmax with novelty bonus. |
| `PacingNeuron` | 5 | Fast/slow cycle-ratio controller. |
| `WellbeingGuardNeuron` | 7 | Escalating intervention: encourage → break → redirect → human. |
| `FairnessNeuron` | 7 | Accommodation-aware latency penalty gate. |

## Processors

The tutoring neurons carry their state directly and operate on domain signals only; no new `ISignalProcessor` implementations are needed beyond those already provided by the affect / curiosity / sleep modules.

## Integration

- **Affect module.** `AffectObservationSignal` is fed to `AmygdalaValenceNeuron` and `AffectIntegrationNeuron`; `AffectStateSignal` in turn drives `FlowStateNeuron` (arousal + valence) and `WellbeingGuardNeuron` (threshold tightening for vulnerable learners, as in the autonomous-AI harm discriminator).
- **Curiosity module.** `NoveltySignal` and `EmpowermentSignal` are additive terms in `ContentSelectionNeuron`'s softmax (`score = zpdFit + β_novelty × novelty`).
- **Sleep module.** When the sleep subsystem is enabled, `HippocampalReplayNeuron` preferentially replays concepts whose `MasteryUpdateSignal` is stale or low-weight, materialising the spaced-repetition schedule that `ForgettingCurveNeuron` only proposes.
- **LLM integration.** Hint text and worked examples can be generated by `LLMKnowledgeNeuron` and verified by `LLMVerificationNeuron` before reaching the learner.

## Configuration

```yaml
tutoring:
  enabled: true
  instance-per-learner: true
  curriculum:
    source: "curriculum.yaml"
    prerequisite-graph: "prereq.json"
  zpd:
    target-success-rate: 0.75
    window-items: 10
  hints:
    max-levels: 3
    delay-between-hints-seconds: 15
  pacing:
    fast-slow-ratio-min: 5
    fast-slow-ratio-max: 20
  wellbeing:
    max-frustration-ticks: 400
    mandatory-break-after-session-minutes: 45
  affect:
    enabled: true
    valence-decay-ticks: 300
  curiosity:
    enabled: true
    beta-novelty: 0.15
  sleep:
    enabled: true
    replay-selects-low-weight: true
  llm:
    enabled: true
    mode: "hints-and-examples"
    verification-strictness: "high"
  fairness:
    accommodation-flags: ["extra-time", "screen-reader", "reduced-animation"]
    response-time-penalty: false
```

## Tests

`TutoringModuleTest` covers:

- Per-signal `ProcessingFrequency` correctness and copy round-trips.
- BKT convergence — 20 correct answers drives `P(know) > 0.8`.
- Forgetting-curve SM-2 interval growth after correct answers, reset on failure.
- Prerequisite graph only returns concepts with all prerequisites mastered.
- ZPD planner picks the candidate nearest the target success rate.
- Hint graduation: META_COGNITIVE → CONCEPTUAL → WORKED_EXAMPLE, null when exhausted.
- Scaffolding fires only on OVERLOAD / FRUSTRATION.
- Pacing slows on overload, speeds on boredom.
- Wellbeing guard escalates through ENCOURAGE → BREAK → REDIRECT → ESCALATE_TO_HUMAN.
- Fairness: accommodation flag disables latency penalty; never modifies `EthicalPriorityNeuron`.
- Integration: OVERLOAD triggers scaffold + eventual wellbeing escalation.
